## SLIDE 2 — Brief About the Idea

Every year, India loses ₹1.52 lakh crore worth of perishable produce in transit — enough food to feed 200 million people. Fruits rot in trucks. Vegetables spoil at depots. Farmers earn nothing. The food just dies.

Why? Because when a refrigerated truck's compressor fails on a highway at 2 AM, there is zero automation. A driver calls a dispatcher. The dispatcher calls a manager. The manager calls a broker. By the time anyone acts, 5,000 kg of mangoes worth ₹3,00,000 are compost.

**Annapurna is an AI-Powered Decision Intelligence Platform** that transforms India's broken cold-chain into an autonomous, self-healing logistics brain. It ingests real-time IoT telemetry, detects cold-chain failures, predicts spoilage 14 days ahead, autonomously reroutes trucks, runs AI-vs-AI price negotiations, and closes emergency sales to nearby wholesalers — all without a single human in the loop.

**Powered by 9 Google Cloud Services:**
🧠 Vertex AI Gemini 2.5 Flash • 🤖 Google ADK Agents • 🔥 Cloud Firestore • 🔍 Firestore Vector Search • 📐 Vertex AI Text Embeddings • 📊 BigQuery + BigQuery ML ARIMA_PLUS • ☁️ Cloud Run • 🌏 Cloud Translate • 📦 Generative AI SDK

This is not a dashboard. This is a Decision Intelligence Engine that converts a ₹3,00,000 total loss into a ₹2,55,000 recovery — in under 90 seconds.

## SLIDE 3 — Solution Explanation

The problem statement asks us to build a platform that ingests data, enables natural language interaction, generates insights and predictions, identifies patterns and anomalies, and supports AI-powered decision-making. **Annapurna does all of this — and goes further by making decisions fully autonomous.**

**How we used Google Cloud to build it:**

**Vertex AI (Gemini 2.5 Flash)** is the AI brain powering 8 distinct capabilities — autonomous cargo routing decisions, adversarial buyer-seller negotiation, camera-based spoilage detection (Vision QC), shipping document extraction (Document AI), natural-language-to-SQL analytics, legal liability reports via RAG, driver voice command processing, and dynamic fleet health metrics. Every call returns structured JSON for reliable real-time parsing.

**Google ADK (Agent Development Kit)** powers our autonomous `fleet_decision_agent` — an `LlmAgent` with real executable function tools: `reroute_truck()` writes a new destination to Firestore, `alert_wholesaler()` dispatches alerts to the wholesaler dashboard. The agent runs via `InMemoryRunner`, reads telemetry, reasons about available markets, and calls these tools autonomously. This is AI that ACTS, not just suggests.

**Cloud Firestore** is our real-time state engine. Every cargo, bid, and transaction syncs instantly across Fleet and Wholesaler dashboards using `onSnapshot` listeners. Bid acceptance uses atomic batch writes — accepting a winning bid, rejecting competitors, and updating cargo quantity all happen in one transaction. Zero polling, zero stale data.

**Firestore Native Vector Search** powers our Legal RAG pipeline. Legal queries are embedded using **Vertex AI Text Embeddings (text-embedding-004)** and matched against FSSAI food safety documents using `findNearest()` with COSINE distance — no Pinecone, no Weaviate, pure Google Cloud.

**BigQuery + BigQuery ML** serves as our telemetry data warehouse. IoT sensor data streams in via `table.insert`. Users ask questions in plain English and Gemini generates SQL that executes against BigQuery — a true Text-to-SQL data agent. Our **ARIMA_PLUS model** predicts spoilage risk 14 days ahead, training and serving predictions inside BigQuery itself — zero separate ML infrastructure.

**Cloud Run** hosts the entire Next.js 15 app as a containerized, auto-scaling service (2 vCPU, 2GB RAM). Push to GitHub → live in 60 seconds.

**Cloud Translate** enables multi-language support across Hindi, Japanese, Thai, Bahasa, and Vietnamese for JAPAC operations.

**What real-world problem does it solve?**

India's cold chain covers only 4% of perishable produce. When a cold chain breaks, the industry standard is total loss — 100% of cargo value gone. Annapurna's autonomous rescue pipeline recovers 70-85% of that value by finding the nearest buyer within the spoilage window. It eliminates the 30-45 minute human decision bottleneck and replaces it with AI action in under 90 seconds.

This directly addresses the hackathon goal: *"Demonstrate how AI, data analytics, and intelligent automation can help create safer, healthier, more efficient, and more sustainable communities."*

**Core workflow: Detect → Reason → Decide → Act → Negotiate → Recover → Learn**

IoT sensors stream temperature, humidity, ethylene, and GPS every 5 seconds → MonitorAgent flags anomalies → ADK DecisionAgent (Gemini 2.5 Flash) evaluates whether cargo can survive transit → Agent calls `reroute_truck()` and `alert_wholesaler()` → Cargo appears on Wholesaler dashboard instantly → NegotiationAgent runs adversarial 3-round pricing → Wholesaler bids → Atomic bid acceptance → Delivery tracking → Payment → All telemetry flows to BigQuery → ARIMA model learns and improves predictions.

## SLIDE 9 — Technologies / Google Services Used

**9 Google Cloud Services — Deeply Integrated, Not Surface-Level**

**1. Vertex AI (Gemini 2.5 Flash)** — The AI brain. Powers 8 capabilities: autonomous routing, adversarial negotiation, vision QC, document extraction, text-to-SQL, legal RAG, voice intent extraction, fleet metrics. Fastest multimodal model — sub-2-second responses. In cold-chain rescue, every second matters.

**2. Google ADK (@google/adk v1.4)** — Autonomous agent framework. Our `fleet_decision_agent` is an `LlmAgent` with real `FunctionTool` bindings that write to Firestore. The ONLY framework that lets an LLM execute real backend functions — not suggest, EXECUTE.

**3. Cloud Firestore** — Real-time database. `onSnapshot` listeners sync all dashboards instantly. Atomic batch writes for bid processing. Sub-100ms writes. When truck rerouted in Mumbai → wholesaler in Pune sees cargo in under 1 second.

**4. Firestore Native Vector Search** — `findNearest()` with COSINE distance for legal RAG retrieval. No Pinecone. No Weaviate. Pure Google Cloud vector search where the data lives.

**5. Vertex AI Text Embeddings (text-embedding-004)** — Generates embeddings for legal queries with RETRIEVAL_QUERY task type. Perfectly paired with Firestore Vector Search.

**6. BigQuery + BigQuery ML (ARIMA_PLUS)** — Telemetry warehouse with streaming inserts. Text-to-SQL via Gemini. 14-day spoilage forecast model trains and serves predictions inside BigQuery — zero MLOps overhead.

**7. Cloud Run** — Production deployment. Auto-scaling containerized Next.js app. 2 vCPU, 2GB RAM. Push → live in 60 seconds. Scales from 0 to 1000+ instances.

**8. Cloud Translate API** — Multi-language support for JAPAC: Hindi, Japanese, Thai, Bahasa, Vietnamese. One API call for any language pair.

**9. Generative AI SDK (@google/genai v2.12)** — Unified AI client with smart proxy pattern. Auto-detects API key vs Vertex AI service account. Single entry point for all 8 Gemini capabilities.

**Why this stack scales:** Firestore auto-scales horizontally. Cloud Run goes from 0 to 1000+ instances. BigQuery handles petabytes without cluster management. ADK agents are stateless — 10,000 trucks processed in parallel.

## SLIDE 11 — Improvements During Prototype Refinement

**Data Layer:** Hardcoded mock data → Live Cloud Firestore with real-time onSnapshot sync across all stakeholders

**AI Engine:** Basic if/else temperature check → Full ADK LlmAgent with Gemini 2.5 Flash + FunctionTool bindings that autonomously execute database writes

**Stakeholders:** Single Fleet dashboard → Multi-stakeholder platform with Fleet + Wholesaler dashboards, real-time bidding, payment, and order lifecycle

**Analytics:** No analytics → Text-to-SQL pipeline (Gemini → BigQuery) + ARIMA_PLUS 14-day spoilage forecast model

**AI Modality:** Text-only → Multi-modal: Vision QC, Document AI, Voice Commands, Text Negotiation — 4 input modalities

**Legal Compliance:** Non-existent → RAG pipeline: text-embedding-004 → Firestore Vector Search (findNearest, COSINE) → Gemini legal reports

**Agent System:** Single function → 5-agent orchestrated system with live Nerve Center terminal

**Deployment:** localhost only → Cloud Run production (2 vCPU, 2GB, auto-scale, globally accessible)

**UX:** Basic HTML → Apple-inspired glassmorphism, Framer Motion animations, dark mode, interactive maps

**Key technical refinements:**
• Atomic bid processing — accept, reject competitors, update quantity in one Firestore batch transaction
• 3-tier ML fallback — ARIMA model → raw aggregation → deterministic pattern. Forecast never fails
• Hybrid AI + Rules engine — if Gemini times out, physics-based fallback kicks in. Demo never breaks
• Smart Vertex client — proxy singleton auto-detects API key vs service account auth
